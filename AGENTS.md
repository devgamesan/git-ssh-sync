# Repository Guidelines

## プロジェクト構成とモジュール配置

このリポジトリは Python 3.12 の CLI プロジェクトで、`src/` レイアウトを採用しています。実行時コードは `src/git_ssh_sync/` にあり、Typer のエントリポイントは `src/git_ssh_sync/cli.py` です。設定、Git コマンド、SSH コマンド、clone 処理、コンソール出力、共通エラーはそれぞれ独立したモジュールに分かれています。テストは `tests/` に置き、`tests/test_git.py` が `src/git_ssh_sync/git.py` を扱うように、対象モジュール名に対応させます。仕様書兼 readme の参照元は `docs/spec.md`、依存関係とビルド設定は `pyproject.toml`、ロックファイルは `uv.lock` です。

## ビルド・テスト・開発コマンド

- `uv sync --dev`: 実行時依存と開発依存をローカル環境にインストールします。
- `uv run pytest`: `pyproject.toml` の設定に従って全テストを実行します。
- `uv run git-ssh-sync --help`: ローカルの CLI を起動し、利用可能なコマンドを確認します。
- `uv build`: 設定済みの `uv_build` バックエンドでパッケージをビルドします。

ローカル実行では、リポジトリ管理下の環境を使うため `uv run ...` を優先してください。

## コーディングスタイルと命名規則

Python 標準のスタイルを基本とし、インデントは 4 スペースにします。公開関数や意図が読み取りにくい値には型注釈を付け、必要な箇所には簡潔なモジュール docstring を置きます。モジュール名と関数名は `snake_case`、クラス名と例外名は `PascalCase` にします。CLI コマンド関数は `init_command` や `clone_command` のような既存パターンに合わせてください。コマンド層は薄く保ち、再利用する処理は補助モジュールへ移します。

## テスト方針

テストフレームワークは pytest です。テストファイルは `test_<module>.py`、テスト関数は `test_<behavior>()` の形式にします。実ユーザー環境や実ネットワークに触れないよう、`tmp_path`、`monkeypatch`、Typer の `CliRunner` を使ってください。Git、SSH、設定、CLI の挙動を変更する場合は、正常系のコマンド構築と、文脈を含むエラーハンドリングの両方をテストします。

## コミットとプルリクエスト

最近のコミットは `Implement clone command layout setup` や `Add uv Python CLI skeleton` のように、短い命令形の要約を使っています。この形式に合わせ、動詞で始め、件名は焦点を絞り、無関係な変更を混ぜないでください。PR にはユーザー向けの変更内容、実行したテストコマンド、関連 issue があればそのリンクを記載します。CLI の表示が変わる場合のみ、必要に応じて端末出力やスクリーンショットを添えます。

## セキュリティと設定の注意

実在する SSH ホスト、ユーザー名、リポジトリ URL、秘密鍵、生成されたローカル設定はコミットしないでください。設定パスを扱うテストでは `HOME` を一時ディレクトリに向け、サブプロセス呼び出しはモックしてください。
